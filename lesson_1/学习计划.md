# 一、4道必考核心题「口语精简背诵版」
## Q1 什么是白盒测试？怎么设计边界条件？
我理解白盒测试就是**不用只看黑盒输入输出**，会先了解内部的实现逻辑、设计限制和资源上限。
设计边界我固定从三个维度入手：
第一，张量Shape的维度范围，比如最小1×1、超大1024×1024、空张量这些边界；
第二，数据类型的取值范围，INT16/INT32、FP16/FP32的最大最小值、极值溢出；
第三，硬件和内存资源限制，比如芯片能承载的最大Tensor尺寸、显存上限。
拿MatMul举例，我会专门测Shape临界、数据类型临界、数值极值临界，把容易出错的边界全覆盖。

## Q2 极限/边界测试属于白盒还是黑盒？
属于**白盒测试思维**。
因为真正的极限和边界，不是靠随机黑盒蒙出来的，必须先清楚它的设计上限、硬件承载能力、算子支持的维度范围，才能精准设计刚好临界、刚好溢出的场景。只靠黑盒随机测，覆盖率很低，测不到真正的隐患点。

## Q3 MatMul、Softmax、LayerNorm分别是什么？怎么测？
- MatMul是矩阵乘法，做特征融合、向量变换；主要测Shape合法性、边界Shape、不同浮点整型兼容性、大数相乘溢出。
- Softmax是概率归一化，输出0到1、总和接近1；重点测极值输入会不会崩、有没有NaN/无穷大、归一化是否正常。
- LayerNorm是层归一化，把数据拉到均值接近0、方差接近1；主要和PyTorch官方结果做精度对齐，看数值稳定性。
我统一方式：用Python构造常规+边界张量，以PyTorch为真值，做功能、精度、边界全覆盖测试。

## Q4 你没有芯片经验，怎么能做算子/芯片测试？
测试底层方法论是通用的，业务变了思路不变。
我把算子当成一个黑盒函数，输入是Tensor、输出是Tensor。
第一步用等价类、边界值梳理三类场景：Shape边界、数据类型边界、数值极值边界；
第二步用Python批量造测试数据、边界用例；
第三步拿PyTorch官方结果当基准，做精度比对、性能比对；
第四步用自动化框架批量回归，每次版本迭代自动对比基线。
我不用懂C++底层开发，专注测试用例设计、自动化、精度校验、回归体系建设，完全能胜任岗位，而且这两天我已经动手写了算子测试Demo，上手很快。

# 二、3个算子可直接运行测试代码（复制就能跑）
## 1. MatMul 精度+性能+边界测试
```python
import torch
import time

# 精度测试
def test_matmul_accuracy():
    a = torch.randn(32, 64)
    b = torch.randn(64, 128)
    res = torch.mm(a, b)
    expect = torch.mm(a, b)
    diff = (res - expect).abs().max()
    assert diff < 1e-5

# 性能测试
def test_matmul_perf():
    a = torch.randn(512, 512)
    b = torch.randn(512, 512)
    start = time.time()
    for _ in range(100):
        torch.mm(a, b)
    cost = time.time() - start
    assert cost < 1.0

# 边界Shape测试
def test_matmul_boundary():
    # 1维边界
    a = torch.randn(1, 1024)
    b = torch.randn(1024, 1)
    res = torch.mm(a, b)
    assert res.shape == (1, 1)

if __name__ == "__main__":
    test_matmul_accuracy()
    test_matmul_perf()
    test_matmul_boundary()
    print("MatMul所有用例执行通过")
```

## 2. Softmax 稳定性+极值测试
```python
import torch

def test_softmax_basic():
    # 极值输入，防止NaN/inf
    x = torch.tensor([1000.0, 1000.0, 1000.0])
    out = torch.softmax(x, dim=0)
    # 无异常值
    assert not torch.isnan(out).any()
    assert not torch.isinf(out).any()
    # 总和接近1
    assert abs(out.sum() - 1.0) < 1e-5

if __name__ == "__main__":
    test_softmax_basic()
    print("Softmax测试通过")
```

## 3. LayerNorm 对齐PyTorch基准
```python
import torch
import torch.nn as nn

def test_layernorm_align():
    x = torch.randn(16, 512)
    ln = nn.LayerNorm(512)
    out = ln(x)
    # 校验均值方差基本符合归一化特性
    mean = out.mean().item()
    assert abs(mean) < 0.1

if __name__ == "__main__":
    test_layernorm_align()
    print("LayerNorm测试通过")
```

# 四、5大高频追问「标准答案（口语版）」
## 追问1 现场设计一个MatMul边界用例怎么设计？
我会从三个维度设计：
第一Shape边界：1×1最小维度、1×1024狭长维度、1024×1024超大矩阵、空张量；
第二数据类型边界：FP16、FP32、INT32不同类型混用；
第三数值边界：全0、负数、极大值，测试会不会溢出、报错、输出异常。
同时校验合法shape正常计算，非法shape友好报错不崩溃。

## 追问2 Softmax输入都是1000极大值，预期是什么？
正常做数值平移避免溢出后，三个输出概率基本均分，接近各三分之一；而且不能出现NaN、无穷大这类异常值，总和要接近1，保证数值稳定性。

## 追问3 性能比基线劣化20%，你会直接拦截吗？
我会先看团队DoD准入标准，如果约定了性能不能明显劣化，就直接拦截；
如果没有明确约定，我不会直接拦，会把版本算子性能拆解对比，输出量化报告，拉研发一起评估是合理波动还是代码引入退化，再定是否准入。

## 追问4 你不会C++，怎么测C++写的算子库？
我不需要写C++底层代码，工作里通过Python绑定调用算子库就行，聚焦功能正确性、精度对齐、边界异常、性能基线测试。
如果出现深层性能瓶颈，我可以用Python工具做链路耗时拆分、给出量化数据，交给C++研发定位底层问题，分工协作完全没问题。

## 追问5 你10年测试最核心的方法论是什么？
核心就是**自动化回归 + 数据驱动做判断**。
不靠主观感觉，固定基准版本、固定环境、固定用例，每次迭代自动回归，精度、性能、异常都用数据对比，有退化自动发现、提前拦截，保障版本质量稳定。
